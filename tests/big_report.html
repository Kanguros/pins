4. Methodology

This analysis employs automated checks to identify potential rule shadowing scenarios. Each check examines specific rule attributes to determine if preceding rules may prevent subsequent rules from being evaluated.

Validation Checks

Check Action

Checks if both rules have the same action (like 'allow' or 'deny'). If the actions are different, the first rule does not fully hide the second one.

Check Application

Checks if the first rule allows all the same applications as the second rule. If the first rule uses 'any' or all the same apps, it can hide the second rule.

Check Services

Checks if the first rule allows all the same network services or ports. If the first rule covers all the same services, it can hide the second rule.

Check Source Zone

Checks if the first rule covers all the same source zones as the second rule. If the first rule uses 'any' or all the same zones, it can hide the second rule.

Check Destination Zone

Checks if the first rule covers all the same destination zones as the second rule. If the first rule uses 'any' or all the same zones, it can hide the second rule.

Check Source Address

Checks if the first rule covers all the same source addresses (like IPs or groups). If the first rule uses 'any' or all the same addresses, it can hide the second rule.

Check Destination Address

Checks if the first rule covers all the same destination addresses. If the first rule uses 'any' or all the same addresses, it can hide the second rule.

Glossary

Rule Shadowing
A condition where a firewall rule is never evaluated because a preceding rule with broader criteria has already matched and processed the traffic.
Device Group
A logical grouping of network devices that share common security policies and rules.
Preceding Rule
A rule that appears earlier in the rule evaluation sequence and may prevent subsequent rules from being processed.
